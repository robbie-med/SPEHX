# SPEX Webapp — Master Build Plan

## Project Goal

Build a static, self-hostable study webapp for the SPEX (Special Purpose Examination) practice test. The user answers 86 multiple-choice clinical vignette questions, receives immediate feedback, sees high-yield explanations, tracks their running score, and can save/reset progress. The app must run identically on `localhost` and on GitHub Pages with zero backend.

---

## Source Material

| File | Description |
|------|-------------|
| `spex-practice-test.pdf` | 86-question SPEX practice test. Block 1: Q1–43 (57 min). Block 2: Q44–86 (57 min). Answer key on page 29. 7 embedded images across 6 pages. |

---

## Constraints

- **Static only.** No server-side code, no database, no build pipeline for the deployed app.
- **GitHub Pages** serves `docs/` folder on `main` branch. All asset paths must be relative.
- **All heavy lifting happens once, locally**, via two Node.js scripts in `scripts/`. Their output (JSON + images) is committed to the repo. The scripts are never run in production.
- **No frontend dependencies** in the deployed app. No npm in `docs/`. Vanilla ES modules only.

## Phase Overview

| Phase | Name | Input | Output | Estimated Time | Blocking? |
|-------|------|-------|--------|----------------|-----------|
| 1 | PDF Extraction | `spex-practice-test.pdf` | `questions.json` (no explanations) + 7 PNGs | 1–2 hrs | Blocks Phase 2 & 3 |
| 2 | Explanation Generation | `questions.json` | `questions.json` (with explanations) | 30 min | Blocks Phase 3 |
| 3 | Frontend App | `questions.json` + images | `index.html`, `style.css`, `app.js` | 2–3 hrs | Blocks Phase 4 |
| 4 | GitHub Pages Deploy | All `docs/` files | Live URL | 15 min | — |

**Critical path:** Phase 1 → Phase 2 → Phase 3 → Phase 4. No parallelism possible.

## Technology Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Frontend framework | None (vanilla JS ES modules) | Solo study tool; React/Vue overhead adds complexity with zero benefit |
| CSS framework | None (plain CSS, one file) | ~200 lines of CSS is simpler than a Tailwind config for this scope |
| PDF text extraction | `pdf-parse` (Node.js) | Simple API, good text output from non-scanned PDFs |
| PDF image extraction | `pdf2pic` + `sharp` (Node.js) | Renders pages to PNG at high DPI; sharp crops to image bounding boxes |
| Explanation source | Claude API (`claude-sonnet-4-6`) | One-time generation; committed to JSON; free after first run |
| Progress persistence | `localStorage` | No account needed; appropriate for single-user local study tool |
| Local dev server | `python3 -m http.server 8080` | Zero install; identical behavior to GitHub Pages |
| GitHub Pages config | `/docs` folder on `main` branch | No GitHub Actions or gh-pages branch needed |
| Bundler | None | The deployed app is three files; a bundler solves no real problem here |

## Success Criteria

- [ ] All 86 questions display with correct text, choices, and answer key
- [ ] All 10 question sets show their shared vignette above each question in the set
- [ ] All 7 images display at the correct questions
- [ ] Answering a question locks the choices and immediately shows correct (green) / incorrect (red) feedback
- [ ] All 5 explanations are revealed after answering (correct answer explanation always shown, incorrect answer explanation shown only for the chosen wrong choice; a "show all" toggle reveals the rest)
- [ ] Score (`X/Y — Z%`) updates after every answer
- [ ] Progress bar updates after every answer
- [ ] Progress survives a page refresh (localStorage)
- [ ] Reset button clears all progress after confirmation
- [ ] Navigation (Prev/Next) works and correctly shows set vignettes regardless of navigation direction
- [ ] App loads and functions correctly at `python3 -m http.server 8080`
- [ ] App loads and functions correctly at `https://[user].github.io/SPEHX/`
- [ ] No console errors in either environment

## Known Risks

| Risk | Likelihood | Mitigation |
|------|-----------|------------|
| `pdf-parse` mangling special characters (Greek letters, subscripts, lab values) | Medium | Post-extraction manual review pass; fix directly in JSON |
| Lab table formatting lost in plain-text extraction | High | Use `<pre>` tags in `questionText` for ~8-10 questions; fix manually in JSON |
| Image crop coordinates incorrect on first pass | Medium | Run `pdf2pic`, open PNGs in image viewer, measure visually, adjust |
| Claude API explanation quality low for image-dependent questions | Low | Provide explicit image descriptions in the prompt; review output before committing |
| Relative path breakage between local and GitHub Pages | Low | Use `./` prefix on all fetch paths; test both environments before marking complete |
